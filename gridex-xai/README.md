# gridex-xai

Small inference pipeline for grid-based audio deepfake XAI.

## Flow

### 1. Prepare inputs with `util/`

`util/make_specs_grid_768.py` converts audio into 768x768 mel-spectrogram PNGs and 4x4 numbered grid overlays. The numbered spectrogram grid image is the only Q1 model input.

Example:

```bash
python util/make_specs_grid_768.py \
  --in-dir /path/to/audio_dir \
  --out-dir /path/to/spec_grid_dir \
  --overwrite
```

or from a CSV:

```bash
python util/make_specs_grid_768.py \
  --pairs-csv /path/to/audio.csv \
  --pairs-audio-col real_path \
  --out-dir /path/to/spec_grid_dir
```

`util/transcript.py` creates the Q2 transcript text with word timing:

```text
[start-end] word [start-end] word ...
```

Example:

```bash
python util/transcript.py \
  --audio /path/to/audio.wav \
  --language en \
  --output-txt /path/to/sample.word_times.txt
```

This transcript is the value for Q2's `{transcript}` placeholder. Current Q2 inference reads transcripts from `META_CSV`, so the generated transcript must be inserted into that CSV's `transcript` column before running Q2. Directly wiring `util/transcript.py` into Q2 is not automated yet.

`util/make_selected_grid_768.py` is only for explanation/display overlays after region IDs are known. It is not part of the model inference pipeline.

### 2. Run Q1 Region Selection

Run:

```bash
bash run_q1_region_selection_inference.sh
```

Q1 uses:

- the numbered spectrogram grid image generated by `util/make_specs_grid_768.py`
- `q1_prompts/q1_system_prompt.txt`
- `q1_prompts/q1_user_prompt.txt`
- `infer_q1_region_selection.py`

Q1 output is written under:

```text
${OUTPUT_BASE_DIR}/${MODEL_TAG}/${RUN_TAG}/
```

The top-level Q1 script merges per-shard sample folders into:

```text
${Q1_OUTPUT_ROOT}
```

By default:

```text
${OUTPUT_BASE_DIR}/${MODEL_TAG}/${RUN_TAG}/merged_for_q2
```

Use the printed `q1_output_root=...` path as Q2's `Q1_OUTPUT_ROOT`.

### 3. Run Q2 Artifact Explanation

Run Q2 after Q1:

```bash
Q1_OUTPUT_ROOT=/path/printed/by/q1/merged_for_q2 \
META_CSV=/path/to/q2_meta_with_transcript.csv \
bash run_q2_artifact_explanation_from_q1.sh
```

Q2 uses:

- the same numbered spectrogram grid image used by Q1
- Q1 region predictions from `Q1_OUTPUT_ROOT`
- transcript text from `META_CSV`
- `q2_prompts/q2_system_prompt.txt`
- `q2_prompts/q2_user_prompt.txt`
- `infer_q2_artifact_explanation_from_q1_json.py`

The Q2 user prompt contains:

```text
Explain the spoof artifact for each of the three selected region IDs in {prompt1_output} . This is the transcript for context: {transcript}
```

`{prompt1_output}` comes from Q1 output. `{transcript}` comes from `META_CSV`. The image path also comes from `META_CSV`; it should point to the same numbered spectrogram grid image used for Q1.

## Files

- `run_q1_region_selection_inference.sh`: user-facing Q1 runner.
- `run_q2_artifact_explanation_from_q1.sh`: user-facing Q2 runner.
- `internal_launchers/`: lower-level shard/GPU launchers used by the top-level scripts.
- `infer_q1_region_selection.py`: Q1 model inference implementation.
- `infer_q2_artifact_explanation_from_q1_json.py`: Q2 model inference implementation.
- `q1_prompts/`: Q1 system/user prompts.
- `q2_prompts/`: Q2 system/user prompts.
- `util/`: data preparation and display utilities.

## Engineer Placeholders

Set or verify these before running.

### Utility Inputs

- `--in-dir` or `--pairs-csv`: source audio for `util/make_specs_grid_768.py`.
- `--out-dir`: destination for spectrogram/grid images.
- `--audio`: source audio for `util/transcript.py`.
- `--output-txt`: optional transcript output path.

### Q1

- `META_JSON`: Q1 metadata JSON. It must point to the Q1 samples and numbered spectrogram grid images.
- `OUTPUT_BASE_DIR`: Q1 output root.
- `Q1_OUTPUT_ROOT`: optional override for the merged Q1 output passed to Q2.
- `MODEL_ID`: Q1 checkpoint. Default is hardcoded to the GRPO-1 checkpoint.
- `SHARD_COUNT`: number of Q1 shards/GPUs.
- `IMAGE_FOLDER`: optional image root used by the lower-level Q1 launcher.
- `CACHE_BASE_DIR`: cache root for vLLM/FlashInfer/HF/Torch caches.

### Q2

- `Q1_OUTPUT_ROOT`: required. Use the Q1 `q1_output_root` printed at the end of Q1.
- `META_CSV`: Q2 CSV with at least `img_path` and `transcript`. `img_path` should point to the same numbered spectrogram grid image used by Q1. `transcript` should be the `[start-end] word` text from `util/transcript.py`.
- `OUTPUT_BASE_DIR`: Q2 output root.
- `MODEL_ID`: Q2 checkpoint. Default is hardcoded to the GRPO-2 Q2 checkpoint.
- `SHARD_COUNT`: number of Q2 shards.
- `GPU_IDS`: comma-separated GPU IDs for Q2, for example `0,1,2,3`.
- `IMAGE_FOLDER`: optional base folder for relative image paths in `META_CSV`.
- `IMG_STEM_CONTAINS`: optional filter for Q2 rows; default is `_LA_D_`.
